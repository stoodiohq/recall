# Recall.team - Frontend & UX Build Plan

**Version:** 1.0  
**Date:** January 9, 2026  
**Status:** Draft

---

## Overview

This document covers everything a user sees and interacts with on recall.team. The backend product plan covers what we build. This covers how users experience it.

### Launch Scope

**At launch:** Claude Code only. One MCP install command.

**Post-launch:** Cursor, Codex, Gemini CLI, and other AI coding tools. Each will need its own integration work.

### Design Principles

| Principle | What It Means |
|-----------|---------------|
| **Fast to value** | User should be working with Recall in under 5 minutes |
| **Show, don't tell** | Demo the product, don't explain it |
| **Minimal decisions** | Every screen has one primary action |
| **Trust signals** | GitHub-native, their data stays theirs |
| **Developer aesthetic** | Clean, fast, no fluff - devs hate fluff |

### Optimization Strategy

| Type | Priority | What It Means |
|------|----------|---------------|
| **LLM Optimized** | Highest | Site content structured for AI discovery. Clear, factual descriptions. Schema markup. LLMs recommending dev tools will find and understand Recall. |
| **SEO Optimized** | High | Standard SEO best practices. Meta tags, semantic HTML, fast load times, mobile-first. Target keywords: "AI coding memory", "team context for AI", "Claude Code memory". |
